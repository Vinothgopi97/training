Constructors, Instance Initialization Block, and Static Block in Java

1. Constructors

A constructor is a special method that is called when an object is created. It has the same name as the class and no return type. 

class Example {
    Example() {
        System.out.println("Constructor called");
    }
    public static void main(String[] args) {
        Example obj = new Example(); // Output: Constructor called
    }
}

2. Instance Initialization Block (init block)

An instance initialization block is a block of code inside a class, enclosed in curly braces { }, that runs every time an object is created, before the constructor. 

class Example {
    {
        System.out.println("Instance init block");
    }
    Example() {
        System.out.println("Constructor");
    }
    public static void main(String[] args) {
        Example obj = new Example();
        // Output:
        // Instance init block
        // Constructor
    }
}

3. Static Block

A static block is a block of code inside a class, prefixed with static, that runs once when the class is loaded into memory (before main or any object is created). 

class Example {
    static {
        System.out.println("Static block");
    }
    Example() {
        System.out.println("Constructor");
    }
    public static void main(String[] args) {
        Example obj1 = new Example();
        Example obj2 = new Example();
        // Output:
        // Static block
        // Constructor
        // Constructor
    }
}

Summary

  • Constructor: Runs when an object is created.
  • Instance init block: Runs before constructor, every time an object is created.
  • Static block: Runs once when the class is loaded.
