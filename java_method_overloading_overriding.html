Method Overloading and Method Overriding in Java

Method Overloading

Method overloading means defining multiple methods with the same name in the same class, but with different parameter lists (different type, number, or order of parameters).

  • Occurs within a single class.
  • Return type can be different or same, but parameter list must be different.
  • Helps increase the readability of the program.
Example: 
class MathUtils {
    int add(int a, int b) {
        return a + b;
    }
    double add(double a, double b) {
        return a + b;
    }
    int add(int a, int b, int c) {
        return a + b + c;
    }
}

MathUtils m = new MathUtils();
System.out.println(m.add(2, 3));        // Output: 5
System.out.println(m.add(2.5, 3.5));    // Output: 6.0
System.out.println(m.add(1, 2, 3));     // Output: 6

Method Overriding

Method overriding means defining a method in a subclass that has the same name, return type, and parameters as a method in its superclass. The subclass method replaces the superclass method when called on a subclass object.

  • Occurs in two classes that have a parent-child (inheritance) relationship.
  • The method in the child class must have the same signature as in the parent class.
  • Used for runtime (dynamic) polymorphism.
  • The @Override annotation is recommended but not required.
Example: 
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}
class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

Animal a = new Animal();
a.sound(); // Output: Animal makes a sound

Dog d = new Dog();
d.sound(); // Output: Dog barks

Animal ad = new Dog();
ad.sound(); // Output: Dog barks (dynamic method dispatch)

Summary

  • Overloading: Same method name, different parameters, same class.
  • Overriding: Same method name and parameters, different classes (inheritance).
