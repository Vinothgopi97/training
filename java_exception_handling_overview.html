Exception Handling in Java

Exception handling in Java is a mechanism to handle runtime errors, so the normal flow of the application can be maintained. It uses try, catch, finally, throw, and throws keywords.

Why Exception Handling?

  • To avoid abnormal termination of the program.
  • To provide meaningful error messages to users.
  • To separate error handling code from regular code.

Basic Syntax

try {
    // Code that may throw an exception
} catch (ExceptionType e) {
    // Code to handle the exception
} finally {
    // Code that will always execute
}

Example

public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int a = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("Caught exception: " + e);
        } finally {
            System.out.println("Finally block executed");
        }
    }
}

Key Points

  • Multiple catch blocks can be used to handle different exception types.
  • The finally block always executes, even if an exception is not caught.
  • Uncaught exceptions cause program termination after finally block.
