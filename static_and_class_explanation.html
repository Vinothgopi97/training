Understanding static, Class, and Instance Concepts

1. static Keyword

The static keyword is used to define class-level members (variables or methods) that are shared among all instances of a class. Static members belong to the class itself, not to any specific object. 

class MyClass {
    static count = 0; // static variable

    static showCount() { // static method
        console.log(MyClass.count);
    }
}

2. Class Variable

A class variable is a variable that is shared by all instances of a class. In many languages, it is defined using the static keyword.

  • Accessed using the class name (e.g., MyClass.count).
  • Changing its value affects all instances.

3. Class Method

A class method is a method that is bound to the class and not the instance. It is usually defined with the static keyword.

  • Called using the class name (e.g., MyClass.showCount()).
  • Cannot access instance variables directly.

4. Instance Variable

An instance variable is a variable that is unique to each object (instance) of a class. 

class MyClass {
    constructor(name) {
        this.name = name; // instance variable
    }
}
  • Accessed using the object (e.g., obj.name).
  • Each object has its own copy.

5. Instance Method

An instance method is a method that operates on instance variables and is called on an object. 

class MyClass {
    constructor(name) {
        this.name = name;
    }
    greet() { // instance method
        console.log("Hello, " + this.name);
    }
}
  • Called using the object (e.g., obj.greet()).
  • Can access instance variables via this.

How Are Static and Instance Members Stored?

Static variables and methods are stored on the class itself. They exist only once in memory, regardless of how many objects (instances) are created from the class. You access them using the class name.

  • Memory for static members is allocated when the class is loaded.
  • All instances share the same static members.
  • Example: MyClass.count or MyClass.showCount()

Instance variables and methods are stored on each object created from the class. Every object gets its own copy of instance variables and can call instance methods.

  • Memory for instance members is allocated when a new object is created.
  • Each object has its own instance variables (e.g., obj.name).
  • Instance methods operate on the data of the specific object.
class MyClass {
    static staticVar = "I belong to the class"; // stored on MyClass
    instanceVar = "I belong to the object";     // stored on each object

    static staticMethod() { /* ... */ }         // stored on MyClass
    instanceMethod() { /* ... */ }              // stored on prototype, called via object
}
let obj1 = new MyClass();
let obj2 = new MyClass();
  • MyClass.staticVar and MyClass.staticMethod() are shared and accessed via the class.
  • obj1.instanceVar and obj2.instanceVar are separate for each object.
  • obj1.instanceMethod() and obj2.instanceMethod() operate on their respective objects.

Usage of static in Inner Classes

In some languages like Java, you can declare an inner class as static. A static inner class does not have access to the instance variables and methods of the outer class. It behaves like a top-level class but is scoped within the outer class. 

// Example in Java
class Outer {
    static class Inner {
        static void display() {
            System.out.println("Static inner class method");
        }
    }
}

// Usage:
Outer.Inner.display();
  • A static inner class can have its own static members.
  • It can be instantiated without an instance of the outer class.
  • It cannot access non-static members of the outer class directly.
