Parallel Streams in Java 8

Parallel streams allow operations on collections to be executed in parallel, leveraging multi-core processors for better performance.

Basic Example:


List<Integer> numbers = Arrays.asList(1,2,3,4,5);
numbers.parallelStream().forEach(System.out::println);

More Examples


// Parallel filtering and mapping
List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David");
List<String> result = names.parallelStream()
    .filter(s -> s.length() > 3)
    .map(String::toUpperCase)
    .collect(Collectors.toList());

// Parallel sum using reduce
int sum = numbers.parallelStream().reduce(0, Integer::sum);

// Comparing sequential and parallel performance
long start = System.currentTimeMillis();
numbers.stream().forEach(n -> {
    try { Thread.sleep(100); } catch (Exception e) {}
});
long sequentialTime = System.currentTimeMillis() - start;

start = System.currentTimeMillis();
numbers.parallelStream().forEach(n -> {
    try { Thread.sleep(100); } catch (Exception e) {}
});
long parallelTime = System.currentTimeMillis() - start;

System.out.println("Sequential: " + sequentialTime);
System.out.println("Parallel: " + parallelTime);

// Preserving order with forEachOrdered
numbers.parallelStream().forEachOrdered(System.out::println);

Note: Parallel streams are best for CPU-intensive tasks and large collections. For small collections, overhead may outweigh benefits.
