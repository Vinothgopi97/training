Constructors in Java

A constructor in Java is a special method that is called when an object is created. It has the same name as the class and does not have a return type (not even void).

  • Used to initialize objects.
  • Called automatically when an object is created using new.
  • If no constructor is defined, Java provides a default constructor.

Types of Constructors

  1. Default Constructor
  2. Parameterized Constructor
  3. Copy Constructor (not built-in, but can be defined by the programmer)

1. Default Constructor

A constructor with no parameters. If you don't define any constructor, Java creates one for you. 

class Student {
    Student() {
        System.out.println("Default constructor called");
    }
}

Student s = new Student(); // Output: Default constructor called

2. Parameterized Constructor

A constructor that takes one or more parameters to initialize the object. 

class Student {
    String name;
    int age;
    Student(String n, int a) {
        name = n;
        age = a;
    }
}

Student s = new Student("John", 20);
System.out.println(s.name); // Output: John
System.out.println(s.age);  // Output: 20

3. Copy Constructor

A constructor that creates a new object by copying the fields of another object of the same class. Java does not provide a built-in copy constructor, but you can define one. 

class Student {
    String name;
    int age;
    Student(String n, int a) {
        name = n;
        age = a;
    }
    // Copy constructor
    Student(Student s) {
        name = s.name;
        age = s.age;
    }
}

Student s1 = new Student("John", 20);
Student s2 = new Student(s1);
System.out.println(s2.name); // Output: John
System.out.println(s2.age);  // Output: 20

Summary

  • Constructors are used to initialize objects.
  • Types: Default, Parameterized, and Copy constructors.
  • Constructor overloading is possible by defining multiple constructors with different parameter lists.

Common Questions about Constructors

Q1: Can I have only a parameterized constructor without a default constructor?

Yes, you can define only a parameterized constructor in your class. However, if you do this, Java does not provide a default (no-argument) constructor automatically. If you try to create an object with no arguments, you will get a compilation error.

Example: 
class Student {
    String name;
    Student(String n) {
        name = n;
    }
}

Student s1 = new Student("John"); // OK
Student s2 = new Student(); // Error: no default constructor

Q2: How to have both default and parameterized constructors?

You can define both constructors in your class. This is called constructor overloading. 

class Student {
    String name;
    Student() {
        name = "Unknown";
    }
    Student(String n) {
        name = n;
    }
}

Student s1 = new Student();        // name = "Unknown"
Student s2 = new Student("John");  // name = "John"

Q3: What happens if I define no constructor at all?

Java automatically provides a default (no-argument) constructor that does nothing extra. 

class Student {
    // No constructor defined
}

Student s = new Student(); // OK, default constructor is used
