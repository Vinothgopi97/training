Stream API in Java 8

The Stream API allows processing collections of objects in a functional style. It supports operations like filter, map, reduce, and more.

Streams can be sequential or parallel, and help write concise and readable code for data manipulation.

Creating Streams

  • Collection.stream() - from collections
  • Arrays.stream(array) - from arrays
  • Stream.of(...) - from values

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
Stream<Integer> stream = numbers.stream();

Intermediate Operations

  • filter(Predicate) - filters elements
  • map(Function) - transforms elements
  • sorted() - sorts elements
  • distinct() - removes duplicates
  • limit(n), skip(n) - restricts elements
  • flatMap(Function) - flattens nested structures

List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Anna");
names.stream()
    .filter(s -> s.startsWith("A"))
    .map(String::toUpperCase)
    .sorted()
    .forEach(System.out::println);

Terminal Operations

  • forEach(Consumer) - performs action for each element
  • collect(Collector) - collects results (e.g., to List, Set, Map)
  • reduce(BinaryOperator) - combines elements
  • count() - counts elements
  • findFirst(), findAny() - finds elements
  • anyMatch(), allMatch(), noneMatch() - checks conditions
  • min(), max() - finds min/max

long count = names.stream().filter(s -> s.length() > 3).count();

Optional<String> firstA = names.stream().filter(s -> s.startsWith("A")).findFirst();

List<String> upperNames = names.stream().map(String::toUpperCase).collect(Collectors.toList());

Reduce Example


int sum = numbers.stream().reduce(0, Integer::sum);

Grouping Example


Map<Integer, List<String>> groupedByLength = names.stream()
    .collect(Collectors.groupingBy(String::length));

Parallel Streams

Use parallelStream() for parallel processing: 


numbers.parallelStream().forEach(System.out::println);

FlatMap Example


List<List<Integer>> listOfLists = Arrays.asList(
    Arrays.asList(1, 2),
    Arrays.asList(3, 4)
);
List<Integer> flatList = listOfLists.stream()
    .flatMap(Collection::stream)
    .collect(Collectors.toList());

Summary

  • Streams do not store data, they operate on source data.
  • Operations are either intermediate (return Stream) or terminal (produce result).
  • Streams support functional-style programming for collections.
