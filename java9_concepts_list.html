Java 9 - Important Concepts

1. Module System (Project Jigsaw)

Java 9 introduced the Java Platform Module System (JPMS), also known as Project Jigsaw. It allows you to modularize your code, improving maintainability and security. Modules explicitly declare their dependencies and the packages they export. 


// module-info.java
module com.example.myapp {
    requires java.base;
    exports com.example.myapp.api;
}

2. JShell (REPL)

JShell is an interactive Read-Eval-Print Loop (REPL) tool for Java. It allows you to quickly test Java code snippets without creating a full class or method. 


jshell> int x = 5;
jshell> System.out.println(x * 2);
10

3. Collection Factory Methods

Java 9 introduced convenient static factory methods for creating immutable lists, sets, and maps. 


List<String> list = List.of("a", "b", "c");
Set<Integer> set = Set.of(1, 2, 3);
Map<String, Integer> map = Map.of("a", 1, "b", 2);

4. Private Interface Methods

Interfaces can now have private methods, which can be used to share code between default and static methods within the interface. 


interface MyInterface {
    private void log(String msg) {
        System.out.println("Log: " + msg);
    }
    default void doSomething() {
        log("Doing something");
    }
}

5. Other Enhancements

  • Stream API Improvements: New methods like takeWhile, dropWhile, and ofNullable were added.
  • Try-With-Resources Enhancement: Effectively final variables can be used in try-with-resources.
  • Diamond Operator for Anonymous Inner Classes: The diamond operator (<>) can now be used with anonymous inner classes.

// Stream API example
List<Integer> nums = List.of(1, 2, 3, 4, 5, 6);
nums.stream().takeWhile(n -> n < 4).forEach(System.out::println); // prints 1, 2, 3

// Try-with-resources example
BufferedReader br = new BufferedReader(new FileReader("file.txt"));
try (br) {
    System.out.println(br.readLine());
}
